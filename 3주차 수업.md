# 📘 C 언어 멘토링 3주차 - 함수, 포인터, 배열

## 1. 함수 (Function)

함수는 **특정 작업을 수행하는 코드 블록**이다.
같은 코드를 반복 작성하지 않고 이름을 붙여 재사용할 수 있다.

### 함수 없이

```c
#include <stdio.h>

int main() {
    int sum1 = 0;
    for(int i = 1; i <= 100; i++) {
        sum1 += i;
    }

    int sum2 = 0;
    for(int i = 1; i <= 100; i++) {
        sum2 += i;
    }

    int sum3 = 0;
    for(int i = 1; i <= 100; i++) {
        sum3 += i;
    }

    printf("%d %d %d\n", sum1, sum2, sum3);
    return 0;
}
```

→ 같은 코드를 3번 반복해서 작성해야 한다.

### 함수 사용

```c
#include <stdio.h>

int sum_1_to_100() {
    int sum = 0;
    for(int i = 1; i <= 100; i++) {
        sum += i;
    }
    return sum;
}

int main() {
    int a = sum_1_to_100();
    int b = sum_1_to_100();
    int c = sum_1_to_100();
    printf("%d %d %d\n", a, b, c);
    return 0;
}
```

→ 함수로 만들어두면 이름만 불러서 몇 번이든 재사용할 수 있다.

---

### 1.1 함수의 구조

```c
반환타입 함수이름(매개변수) {
    // 실행할 코드
    return 반환값;
}
```

예시:

```c
int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(3, 5);
    printf("%d\n", result);  // 8
    return 0;
}
```

| 구성 요소 | 의미 |
|-----------|------|
| `int` (앞) | 반환 타입 — 함수가 돌려주는 값의 타입 |
| `add` | 함수 이름 |
| `int a, int b` | 매개변수 — 함수가 받는 입력값 |
| `return a + b` | 반환값 — 호출한 곳으로 돌려주는 값 |

---

### 1.2 매개변수와 인자

- **매개변수(parameter)**: 함수 정의에서 선언하는 변수 (`int a, int b`)
- **인자(argument)**: 함수를 호출할 때 실제로 전달하는 값 (`3, 5`)

```c
int add(int a, int b) { ... }  // a, b는 매개변수
add(3, 5);                     // 3, 5는 인자
```

---

### 1.3 반환값과 void

반환할 값이 없을 때는 반환 타입을 `void`로 지정한다.

```c
void print_hello() {
    printf("Hello!\n");
    // return 불필요
}

int main() {
    print_hello();  // Hello!
    return 0;
}
```

---

### 1.4 함수와 스택(Stack) 동작

함수가 호출될 때, 메모리의 **스택(stack)** 영역에 해당 함수의 공간(스택 프레임)이 생성된다.  
함수가 종료되면 그 공간은 **자동으로 사라진다**.

> **핵심** — 함수에 값을 넘기면 **복사본**이 만들어진다.  
> 함수 안에서 매개변수를 바꿔도 원본에는 영향이 없다.

```c
void change(int x) {
    x = 100;  // 복사본만 바뀜
}

int main() {
    int a = 10;
    change(a);
    printf("%d\n", a);  // 10 — 원본 그대로
    return 0;
}
```

이 한계를 극복하기 위해 **포인터**를 사용한다 (4.3 참고).

---

### 1.5 지역변수와 전역변수

변수는 **선언된 위치**에 따라 접근할 수 있는 범위(스코프)가 달라진다.

```c
int global = 100;  // 전역변수 — 파일 어디서든 접근 가능

void foo() {
    int local = 10;  // 지역변수 — foo() 안에서만 유효
    printf("%d %d\n", global, local);
}

int main() {
    printf("%d\n", global);  // OK
    // printf("%d\n", local); // 오류 — main에서는 접근 불가
    foo();
    return 0;
}
```

| 구분 | 선언 위치 | 생존 기간 | 초기값 |
|------|-----------|-----------|--------|
| 지역변수 | 함수 내부 | 함수 실행 중에만 | 쓰레기값 (초기화 필수) |
| 전역변수 | 함수 외부 | 프로그램 종료까지 | 자동으로 0 |

> **주의** — 전역변수는 어디서든 바꿀 수 있어 편리하지만,  
> 코드가 커질수록 어디서 값이 바뀌는지 추적하기 어려워진다.  
> 꼭 필요한 경우가 아니면 **지역변수를 우선** 사용하는 것이 좋다.

---

### 1.6 값에 의한 전달 vs 주소에 의한 전달

C에서 함수에 값을 넘기는 방법은 두 가지다.

**값에 의한 전달 (Call by Value)** — 복사본을 넘긴다

```c
void double_val(int x) {
    x = x * 2;  // 복사본만 바뀜
}

int n = 5;
double_val(n);
printf("%d\n", n);  // 5 — 원본 그대로
```

**주소에 의한 전달 (Call by Reference)** — 주소를 넘겨 원본을 조작한다

```c
void double_val(int *x) {
    *x = *x * 2;  // 원본을 직접 수정
}

int n = 5;
double_val(&n);
printf("%d\n", n);  // 10 — 원본이 바뀜
```

| | 값 전달 | 주소 전달 |
|--|---------|-----------|
| 넘기는 것 | 값의 복사본 | 변수의 주소 |
| 원본 변경 | 불가 | 가능 |
| 언제 쓰나 | 단순 계산, 읽기 전용 | 원본을 바꾸거나 여러 값을 돌려줄 때 |

---

### 1.7 재귀 함수 (Recursion)

함수가 **자기 자신을 호출**하는 것을 재귀라고 한다.  
반드시 **종료 조건(base case)** 이 있어야 무한 호출을 막을 수 있다.

```c
int factorial(int n) {
    if (n == 0) return 1;      // 종료 조건
    return n * factorial(n - 1); // 자기 자신 호출
}

int main() {
    printf("%d\n", factorial(5));  // 120
    return 0;
}
```

호출 흐름:

```
factorial(5)
  → 5 * factorial(4)
       → 4 * factorial(3)
            → 3 * factorial(2)
                 → 2 * factorial(1)
                      → 1 * factorial(0)
                           → 1  (종료)
```

> **주의** — 재귀는 호출될 때마다 스택 프레임이 쌓인다.  
> 종료 조건이 없거나 잘못되면 스택이 넘쳐 **스택 오버플로우(stack overflow)** 가 발생한다.

---

### 1.8 함수 선언

함수를 `main()` 아래에 정의하면 컴파일러가 인식하지 못한다.  
이를 해결하기 위해 **상단에 선언(프로토타입)** 을 미리 적는다.

```c
int add(int a, int b);  // 선언 (세미콜론 주의)

int main() {
    printf("%d\n", add(3, 5));  // 문제 없음
    return 0;
}

int add(int a, int b) {  // 정의
    return a + b;
}
```

---

### 1.9 함수 설계 원칙

좋은 함수는 **한 가지 일만** 한다.

```c
// ❌ 나쁜 예 — 입력, 계산, 출력을 한 함수에 다 넣음
void do_everything() {
    int a, b;
    scanf("%d %d", &a, &b);
    int result = a + b;
    printf("%d\n", result);
}

// ✅ 좋은 예 — 역할을 분리
int add(int a, int b) {
    return a + b;
}

int main() {
    int a, b;
    scanf("%d %d", &a, &b);
    printf("%d\n", add(a, b));
}
```

함수를 잘 나누면 **코드를 읽기 쉽고**, **수정할 때 영향 범위가 줄어든다**.

## 2. 메모리와 주소

프로그램에서 모든 변수는 메모리 공간에 저장된다.  
각 변수는 고유한 주소(address)를 가진다.

```c
#include <stdio.h>

int main() {
    int a = 10;
    printf("%p\n", &a); // a의 주소 출력
    return 0;
}
```

- a → 변수 a에 저장된 값  
- &a → 변수 a가 저장되어있는 주소
> **참고** — 주소 값은 실행 환경마다 다르게 출력된다. 중요한 것은 '어디에 있는가'라는 위치 개념이다.

---

## 3. 포인터 (Pointer)

포인터는 **다른 변수의 주소를 저장하는 변수**이다.  
`*`를 붙여 선언하며, 타입은 가리킬 변수의 타입과 맞춰야 한다.

```c
int  *p;   // int형 변수의 주소를 저장하는 포인터
char *c;   // char형 변수의 주소를 저장하는 포인터
```

---

### 3.1 포인터 초기화

```c
int a = 10;
int *p = &a;  // p에 a의 주소를 저장
```

| 표현 | 의미 |
|------|------|
| `p` | a의 주소 (예: 0x1A3F) |
| `*p` | p가 가리키는 곳의 값 → a의 값(10) |

### 3.2 역참조 (Dereference)

`*` 연산자를 포인터 앞에 붙이면 그 주소에 저장된 값에 접근할 수 있다.  
값을 읽는 것은 물론, 직접 수정도 가능하다.

```c
int a = 10;
int *p = &a;

printf("%d\n", *p);  // 10 — a의 값을 읽음

*p = 20;             // a의 값을 20으로 변경
printf("%d\n", a);   // 20 — 원본 변수가 바뀜
```

> **비유** — `p`는 우체통 번호, `*p`는 그 우체통 안에 들어 있는 편지다.  
> 번호를 알면(p) 편지를 꺼내거나(*p) 새 편지로 교체(*p = 새값)할 수 있다.

---

## 4. 포인터 활용 — swap 함수

함수 안에서 두 변수를 교환하려면 **값이 아니라 주소를 넘겨야** 원본이 바뀐다.

### 4.1 잘못된 예 (값을 넘기는 경우)

```c
void swap_wrong(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
    // 지역 변수만 바뀌고, 호출한 쪽의 x, y는 그대로
}

swap_wrong(x, y);  // x, y 변화 없음
```

### 4.2 올바른 예 (주소를 넘기는 경우)

```c
void swap(int *a, int *b) {
    int temp = *a;  // a가 가리키는 값을 임시 저장
    *a = *b;        // b의 값을 a 자리에 씀
    *b = temp;      // 임시 저장값을 b 자리에 씀
}

int x = 1, y = 2;
swap(&x, &y);       // 주소를 넘김

printf("%d %d\n", x, y);  // 2 1 — 원본이 바뀜
```

> 이 패턴은 `scanf`에서 `&`를 쓰는 이유와 동일하다.  
> `scanf("%d", &n)` — 함수 안에서 n의 값을 직접 채워 넣기 위해 주소를 넘긴다.

## 5. 배열 (Array)

```c
int arr[3] = {1, 2, 3};
```

배열은 연속된 메모리 공간이다.

---

### 5.1 배열과 주소

```c
printf("%p\n", arr);
printf("%p\n", &arr[0]);
```

→ 결과 동일

---

### 5.2 배열과 포인터 관계

```c
arr[i] == *(arr + i)
```

예:

```c
printf("%d\n", arr[1]);
printf("%d\n", *(arr + 1));
```

---

### 5.3 포인터로 배열 다루기

```c
int *p = arr;

printf("%d\n", p[1]);
printf("%d\n", *(p + 1));
```

## 10. 핵심 정리

- 함수: 재사용 가능한 코드 블록, 호출 시 스택에 프레임 생성  
- 포인터: 주소를 저장하는 변수  
- 배열: 연속된 메모리 공간  
- 동적할당: 실행 중 메모리 생성  
- free: 메모리 해제

## 11. 한줄 정리

함수는 코드를 묶어 재사용하는 단위이며,  
포인터를 함수에 넘기면 원본 데이터를 직접 조작할 수 있다.  
배열은 포인터를 기반으로 동작하며,  
동적할당은 그 포인터를 통해 메모리를 직접 사용하는 방법이다.
