## 1. 변수와 자료형

### 1.1 변수란?

변수는 **값을 저장하는 이름 붙은 메모리 공간**이다.

```c
int age = 20;       // 정수
double pi = 3.14;   // 실수
char grade = 'A';   // 문자 하나
```

### 1.2 자료형 크기와 범위

| 타입 | 크기 | 범위 | 주요 용도 |
|------|------|------|-----------|
| `char` | 1바이트 | -128 ~ 127 | 문자, 작은 정수 |
| `int` | 4바이트 | 약 -21억 ~ 21억 | 일반 정수 |
| `long long` | 8바이트 | 약 ±9.2 × 10¹⁸ | 큰 정수 |
| `float` | 4바이트 | 소수 6~7자리 | 실수 (낮은 정밀도) |
| `double` | 8바이트 | 소수 15~16자리 | 실수 (높은 정밀도) |
| `unsigned int` | 4바이트 | 0 ~ 약 42억 | 음수 없는 정수 |

> **정수는 `int`, 실수는 `double`** 을 기본으로 사용한다.

### 1.3 변수 선언과 초기화

```c
int a;          // 선언만 — 쓰레기값이 들어 있음!
int b = 0;      // 선언 + 초기화
int c, d = 5;   // c는 쓰레기값, d는 5
```

> **주의** — 초기화하지 않은 변수를 그냥 쓰면 예측 불가능한 값이 나온다.  
> 항상 선언과 동시에 초기화하는 습관을 들이자.

## 2. printf / scanf 서식 지정자

### 2.1 기본 서식

| 서식 | 타입 | 예시 |
|------|------|------|
| `%d` | `int` | `printf("%d", 42)` → `42` |
| `%lld` | `long long` | `printf("%lld", 1234567890LL)` |
| `%f` | `double` | `printf("%f", 3.14)` → `3.140000` |
| `%c` | `char` | `printf("%c", 'A')` → `A` |
| `%s` | 문자열 | `printf("%s", "hi")` → `hi` |
| `%p` | 주소 | `printf("%p", &a)` |

### 2.2 출력 폭과 정렬

```
%[플래그][최소 폭][.정밀도]타입
```

```c
printf("%5d\n",   42);   //    42  (폭 5, 오른쪽 정렬)
printf("%-5d|\n", 42);   // 42   | (폭 5, 왼쪽 정렬)
printf("%05d\n",  42);   // 00042 (0으로 빈칸 채우기)
```

### 2.3 실수 자릿수 지정

```c
double pi = 3.14159265;

printf("%f\n",    pi);   // 3.141593  (기본: 소수 6자리)
printf("%.2f\n",  pi);   // 3.14      (소수 2자리)
printf("%.0f\n",  pi);   // 3         (소수점 없음)
```

> **비유** — 폭은 액자 크기, 정밀도는 그림의 해상도다.  
> 값이 액자보다 작으면 빈 공간(공백 또는 0)이 채워진다.

### 2.4 자주 쓰는 서식 한눈에 보기

| 서식 | 설명 | 출력 예 (`n=42`) |
|------|------|------------------|
| `%d` | 정수 그대로 | `42` |
| `%5d` | 폭 5, 오른쪽 정렬 | `   42` |
| `%-5d` | 폭 5, 왼쪽 정렬 | `42   ` |
| `%05d` | 폭 5, 0 채우기 | `00042` |
| `%.2f` | 소수 2자리 | `3.14` |
| `%8.2f` | 폭 8 + 소수 2자리 | `    3.14` |

## 3. 연산자

### 3.1 산술 연산자

```c
int a = 10, b = 3;

printf("%d\n", a + b);  // 13
printf("%d\n", a - b);  // 7
printf("%d\n", a * b);  // 30
printf("%d\n", a / b);  // 3   ← 정수 나눗셈! 소수점 버림
printf("%d\n", a % b);  // 1   ← 나머지
```

> **핵심** — `int / int`는 항상 정수 결과다.  
> `10 / 3`은 `3.333...`이 아니라 `3`이다.

### 3.2 대입 연산자

```c
int a = 5;
a += 3;   // a = a + 3  → 8
a -= 2;   // a = a - 2  → 6
a *= 4;   // a = a * 4  → 24
a /= 6;   // a = a / 6  → 4
a %= 3;   // a = a % 3  → 1
```

### 3.3 비교 연산자 (결과는 0 또는 1)

```c
int a = 5, b = 3;

a == b   // 0 (같지 않으므로 거짓)
a != b   // 1 (다르므로 참)
a >  b   // 1
a <  b   // 0
a >= b   // 1
a <= b   // 0
```

### 3.4 논리 연산자

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `&&` | AND — 둘 다 참이어야 참 | `(a > 0 && b > 0)` |
| `\|\|` | OR — 하나라도 참이면 참 | `(a > 0 \|\| b > 0)` |
| `!` | NOT — 참/거짓 반전 | `!(a == b)` |

### 3.5 증감 연산자

```c
int a = 5;
a++;    // a = 6 (후위: 현재값 사용 후 증가)
++a;    // a = 7 (전위: 먼저 증가 후 사용)
a--;    // a = 6
```

## 4. 조건문

### 4.1 if / else if / else

```c
int score = 85;

if (score >= 90) {
    printf("A\n");
} else if (score >= 80) {
    printf("B\n");   // 여기 출력
} else if (score >= 70) {
    printf("C\n");
} else {
    printf("F\n");
}
```

### 4.2 switch

```c
int menu = 2;

switch (menu) {
    case 1:
        printf("더하기\n");
        break;
    case 2:
        printf("빼기\n");  // 여기 출력
        break;
    case 3:
        printf("곱하기\n");
        break;
    default:
        printf("잘못된 입력\n");
        break;
}
```

> **주의** — `break`를 빠뜨리면 아래 case까지 계속 실행된다 (fall-through).

## 5. 반복문

### 5.1 for

```c
for (초기화; 조건; 증감) {
    // 반복할 코드
}
```

```c
for (int i = 0; i < 5; i++) {
    printf("%d ", i);  // 0 1 2 3 4
}
```

### 5.2 while

조건이 참인 동안 반복한다.

```c
int i = 0;
while (i < 5) {
    printf("%d ", i);  // 0 1 2 3 4
    i++;
}
```

### 5.3 do-while

**최소 한 번**은 실행한 뒤 조건을 검사한다.

```c
int i = 0;
do {
    printf("%d ", i);  // 0 1 2 3 4
    i++;
} while (i < 5);
```

### 5.4 break / continue

```c
for (int i = 0; i < 10; i++) {
    if (i == 3) continue;   // 3은 건너뜀
    if (i == 7) break;      // 7에서 반복 종료
    printf("%d ", i);       // 0 1 2 4 5 6
}
```

### 5.5 중첩 반복문

```c
for (int i = 1; i <= 3; i++) {
    for (int j = 1; j <= 3; j++) {
        printf("%d*%d=%d  ", i, j, i*j);
    }
    printf("\n");
}
```

출력:

```
1*1=1  1*2=2  1*3=3
2*1=2  2*2=4  2*3=6
3*1=3  3*2=6  3*3=9
```

## 6. 함수

### 6.1 함수의 구조

```c
반환타입 함수이름(매개변수) {
    // 실행할 코드
    return 반환값;
}
```

```c
int add(int a, int b) {
    return a + b;
}

int main() {
    int result = add(3, 5);
    printf("%d\n", result);  // 8
    return 0;
}
```

| 구성 요소 | 의미 |
|-----------|------|
| `int` (앞) | 반환 타입 |
| `add` | 함수 이름 |
| `int a, int b` | 매개변수 |
| `return a + b` | 반환값 |

### 6.2 매개변수 vs 인자

```c
int add(int a, int b) { ... }  // a, b → 매개변수 (정의 시)
add(3, 5);                     // 3, 5 → 인자 (호출 시)
```

### 6.3 void 함수

반환값이 없을 때는 `void`를 사용한다.

```c
void greet(char name[]) {
    printf("안녕하세요, %s님!\n", name);
}

int main() {
    greet("철수");  // 안녕하세요, 철수님!
    return 0;
}
```

### 6.4 값에 의한 전달 vs 주소에 의한 전달

함수에 값을 넘기면 **복사본**이 만들어지므로 원본이 바뀌지 않는다.

```c
void change(int x) {
    x = 100;          // 복사본만 바뀜
}

int main() {
    int a = 10;
    change(a);
    printf("%d\n", a);  // 10 — 원본 그대로
    return 0;
}
```

원본을 바꾸려면 **주소(포인터)** 를 넘겨야 한다.

```c
void change(int *x) {
    *x = 100;         // 원본 직접 수정
}

int main() {
    int a = 10;
    change(&a);
    printf("%d\n", a);  // 100
    return 0;
}
```

### 6.5 지역변수 vs 전역변수

| 구분 | 선언 위치 | 생존 기간 | 초기값 |
|------|-----------|-----------|--------|
| 지역변수 | 함수 내부 | 함수 실행 중에만 | 쓰레기값 |
| 전역변수 | 함수 외부 | 프로그램 종료까지 | 자동으로 0 |

```c
int global = 100;  // 전역변수

void foo() {
    int local = 10; // 지역변수
    printf("%d %d\n", global, local);
}
```

### 6.6 함수 프로토타입 (선언)

`main()` 아래에 함수를 정의하면 컴파일러가 인식 못 한다.  
**상단에 미리 선언**해 해결한다.

```c
int add(int a, int b);   // 선언 (세미콜론 필수)

int main() {
    printf("%d\n", add(3, 5));
    return 0;
}

int add(int a, int b) {  // 정의
    return a + b;
}
```

### 6.7 재귀 함수

함수가 자기 자신을 호출하는 것.  
반드시 **종료 조건(base case)** 이 있어야 한다.

```c
int factorial(int n) {
    if (n == 0) return 1;        // 종료 조건
    return n * factorial(n - 1); // 재귀 호출
}

// factorial(3) → 3 * factorial(2)
//                     → 2 * factorial(1)
//                          → 1 * factorial(0)
//                               → 1
// 결과: 3 * 2 * 1 * 1 = 6
```

> 종료 조건이 없으면 스택이 넘쳐 **스택 오버플로우**가 발생한다.

## 7. 오버플로우 (Overflow)

### 7.1 왜 발생하는가?

각 자료형은 저장할 수 있는 비트 수가 정해져 있다.  
그 범위를 벗어나는 값을 저장하면 **값이 반대쪽 끝으로 돌아간다(wrap-around)**.

```c
int max = 2147483647;    // int 최댓값
printf("%d\n", max);     // 2147483647
printf("%d\n", max + 1); // -2147483648  ← 오버플로우!
```

> **비유** — 자동차 주행 거리계가 999,999에서 1 더 가면 000,000이 되는 것과 같다.

### 7.2 오버플로우 예방

큰 숫자가 예상될 때는 `long long`을 사용하고 `%lld`로 출력한다.

```c
int a = 100000, b = 100000;

// ❌ int끼리 곱셈 — 오버플로우
int wrong = a * b;
printf("%d\n", wrong);   // 이상한 값

// ✅ long long으로 계산
long long correct = (long long)a * b;
printf("%lld\n", correct);  // 10000000000
```

> 실수에서 범위를 넘으면 `inf`(무한대)가 출력된다.
> ```c
> float f = 3.4e38 * 10.0f;
> printf("%f\n", f);  // inf
> ```

## 8. 캐스팅 (Type Casting)

캐스팅은 **변수의 타입을 다른 타입으로 강제 변환**하는 것이다.

### 8.1 자동 형 변환 (묵시적)

작은 타입이 큰 타입과 연산되면 자동으로 변환된다.

```c
int a = 5;
double b = 2.0;
double result = a + b;  // a가 double로 자동 변환
printf("%f\n", result); // 7.000000
```

### 8.2 강제 형 변환 (명시적)

`(타입)값` 형태로 직접 변환을 지시한다.

```c
int a = 7, b = 2;

double r1 = a / b;           // 3.000000 ← 정수 나눗셈 먼저!
double r2 = (double)a / b;   // 3.500000 ← 캐스팅 후 나눗셈

printf("%.1f\n", r1);  // 3.0
printf("%.1f\n", r2);  // 3.5
```

> **핵심** — `int / int`는 항상 정수 나눗셈이다.  
> 소수점 결과를 원하면 **피연산자 중 하나를** `(double)`로 캐스팅한다.

### 8.3 큰 타입 → 작은 타입 주의

```c
double d = 3.99;
int i = (int)d;      // 3  ← 반올림이 아니라 소수점 이하 버림!

int big = 300;
char c = (char)big;  // char 범위 초과 → 의미없는 값
```

### 8.4 캐스팅 방향 요약

| 변환 방향 | 결과 | 안전? |
|-----------|------|-------|
| `int` → `double` | 소수점 추가 | ✅ 안전 |
| `double` → `int` | 소수점 버림 | ⚠️ 손실 |
| `int` → `long long` | 값 유지 | ✅ 안전 |
| `long long` → `int` | 초과분 잘림 | ⚠️ 손실 |
| `int / int` | 정수 결과 | ⚠️ 주의 |
| `(double)int / int` | 실수 결과 | ✅ 의도대로 |

### 8.5 캐스팅은 연산 전에!

```c
int a = 100000, b = 50000;

// ❌ 곱셈 후 캐스팅 — 이미 오버플로우된 값을 변환
double wrong = (double)(a * b);
printf("%f\n", wrong);   // 오버플로우된 값이 그대로 실수로 변환됨

// ✅ 곱셈 전에 캐스팅 — 연산 자체가 실수로 진행됨
double correct = (double)a * b;
printf("%f\n", correct); // 5000000000.000000
```

> 캐스팅은 **연산 전에** 해야 효과가 있다.  
> 결과가 나온 뒤 캐스팅하면 이미 잘린 값이 변환된다.

## 9. 핵심 정리

| 주제 | 핵심 한 줄 |
|------|------------|
| 변수 | 이름 붙은 메모리 공간 — 반드시 초기화할 것 |
| 자료형 | 크기와 범위가 다름 — 상황에 맞게 선택 |
| printf 서식 | `%폭.정밀도타입` — 출력 형태를 세밀하게 제어 |
| 연산자 | `int / int`는 정수 결과 — 소수 결과를 원하면 캐스팅 |
| 조건문 | `if-else if-else`, `switch` — `break` 빠뜨리지 말 것 |
| 반복문 | `for` / `while` / `do-while` — `break`, `continue`로 흐름 제어 |
| 함수 | 한 가지 일만 하도록 설계 — 프로토타입 선언 주의 |
| 오버플로우 | 범위 초과 시 반대쪽 끝으로 돌아감 — `long long` 사용 |
| 캐스팅 | 연산 **전에** 적용 — 작은 타입으로의 변환은 손실 발생 가능 |
